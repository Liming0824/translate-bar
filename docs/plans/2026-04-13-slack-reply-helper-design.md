# Slack Reply Helper — Design

**Date:** 2026-04-13
**Status:** Approved

## Problem

Composing casual English responses in Slack is slow when you think in Chinese first. Going back and forth with a translator or AI to find the right phrasing breaks flow. A quick, hotkey-driven reply helper integrated into TranslateBar would eliminate this friction.

## Solution

Extend TranslateBar with a second hotkey (`Cmd+Shift+R`) that opens a reply-composition panel. The user selects a Slack message, triggers the hotkey, types their intent (in Chinese or rough English), and gets a natural English response generated by Claude API. The response auto-copies to clipboard for immediate pasting.

## User Flow

1. Select a Slack message
2. Press `Cmd+Shift+R`
3. ReplyPanel appears near cursor showing:
   - The selected message (context)
   - A text input for the user's intent (Chinese or English)
   - A "Generate" button (or press Enter)
4. Claude generates a casual English response
5. Response auto-copies to clipboard
6. User pastes into Slack

## Architecture

### New Files

| File | Purpose |
|------|---------|
| `Services/ClaudeService.swift` | HTTP client for Claude Messages API. Sends the selected message + user intent, returns a casual English reply. |
| `UI/ReplyPanel.swift` | Floating panel with: selected message display, text input for intent, generated response display, auto-copy behavior. |

### Modified Files

| File | Change |
|------|--------|
| `Utilities/HotkeyManager.swift` | Support registering a second hotkey with a separate callback. |
| `App/AppDelegate.swift` | Add `handleHotkeyReply()` flow: get selected text, open ReplyPanel, wire up ClaudeService. |
| `UI/SettingsView.swift` | Add Claude API key field, stored in Keychain under separate account (`claude-api-key`). |

### Unchanged Files

- `MenuBarController.swift` — no changes needed
- `TranslationPanel.swift` — existing translate flow untouched
- `TranslationService.swift`, `LanguageDetector.swift` — unchanged
- `KeychainHelper.swift`, `AccessibilityHelper.swift` — reused as-is

## Claude API Integration

### API Call

Use the Claude Messages API (`POST https://api.anthropic.com/v1/messages`) with:
- Model: `claude-haiku-4-5-20251001` (fast, cheap, good enough for small talk)
- Max tokens: ~150 (keep responses brief)
- System prompt tuned for casual conversation

### System Prompt

```
You are helping someone reply in a casual Slack conversation. Generate a natural, friendly English response. The user will provide:
1. The message they are replying to
2. What they want to say (may be in Chinese or rough English)

Interpret their meaning and produce a native-sounding English reply. Keep it brief, conversational, and warm. This is off-topic small talk, not work discussion.
```

### User Message Format

```
Message I'm replying to: "{selected_text}"

What I want to say: "{user_intent}"
```

## ReplyPanel UI

```
┌─────────────────────────────────────┐
│ Replying to:                        │
│ "hey anyone tried that new ramen    │
│  place on 3rd st?"                  │
│                                     │
│ What do you want to say?            │
│ ┌─────────────────────────────────┐ │
│ │ 太好吃了我去过                    │ │
│ └─────────────────────────────────┘ │
│                     [Generate ↵]    │
│                                     │
│ "Oh yeah I've been there, it's so   │
│  good!"                             │
│                                     │
│              ✓ Copied to clipboard  │
└─────────────────────────────────────┘
```

- Panel width: ~400px (consistent with TranslationPanel)
- Floating, near cursor, same dismiss behavior (Esc, click outside)
- Text input is focused on panel open so user can type immediately
- Enter in text field triggers generation
- Response auto-copies to clipboard on generation
- Panel stays open briefly so user can read the response

## Key Decisions

- **Stateless** — no conversation history, each invocation is independent
- **No Slack API integration** — uses system clipboard/accessibility to get selected text, same as translate mode
- **Separate API key** — Claude key stored independently from Google Translate key in Keychain
- **Haiku model** — fast response time is more important than sophistication for small talk
- **Auto-copy** — reduces friction to a single paste after generation
